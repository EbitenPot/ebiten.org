Ebiten 2.2 Release Notes

v2.2.0

Issues for v2.2.0

Note that v2.2.0 is not released yet and this document is a draft.

Nintendo Switch™

Ebiten now supports Nintendo Switch. For more information, please see Nintendo Developer Portal, the portal site for Nintendo platform developers.

Audio reimplementation

(#1549)

The audio implementaiton is much improved to have a bigger buffer for each player. Delays and noises are reduced.

Standard gamepad layout mappings

(#1557)

Until v2.1, Ebiten's gamepad button and axis assignments varied among gamepads and environments. For example, a gamepad button number 1 might indicate a different physical button in another environment. It was hard to keep poratbility with gamepads.

Ebiten v2.2 adopted the standard gamepad layout in the Web standard. The standard gamepad is a model for modern gamepads like Xbox, PlayStation and Nintendo Switch. The standard gamepad assumes these buttons or axes:

  • Two sticks
  • Four buttons on the left hand (especially for arrows or D-pad)
  • Four buttons on the right hand
  • Four buttons on the front
  • Three buttons on the center

If a standard gamepad layout is available for your gamepad in your environment, you can use APIs for standard gamepads like StandardGameButton. You can use constant values for buttons or axes for almost any gamepads and environments, and don't have to worry about the button/axis assignment differences.

On desktops, SDL's gamepad database is used. On browsers, browsers' standard mappings are used. On mobiles, standard mappings are not available yet.

  • type ebiten.StandardGamepadAxis
  • type ebiten.StandardGamepadButton
  • ebiten.IsStandardGamepadButtonPressed
  • ebiten.IsStandardGamepadLayoutAvailable
  • ebiten.StandardGamepadAxisValue
  • ebiten.StandardGamepadButtonValue
  • ebiten.UpdateStandardGamepadLayoutMappings

For actual usages, see examples/gamepad.

Vector graphics

(#1684)

Vector graphics APIs in Ebiten were redesigned and reimplemented with stencil buffers.

A boolean member FillMode is added to ebiten.DrawTrianglesOptions. The fill mode ebiten.EvenOdd indicates the rule how an overlapped region is rendered.

vector package now has these functions:

  • (*vector.Path).AppendVerticesAndIndicesForFilling
  • (*vector.Path).Arc
  • (*vector.Path).ArcTo
  • (*vector.Path).CubicTo
  • (*vector.Path).LineTo
  • (*vector.Path).MoveTo
  • (*vector.Path).QuadTo

For actual usages, see examples/vector.

macOS's native fullscreen

(#1506)

On macOS, now fullscreening a window creates a new independent space and fit the window with the monitor.

Handling window-closing

(#1574)

  • ebiten.IsWindowBeingClosed: returns true when the user is trying to close the window on desktops.
  • ebiten.IsWindowClosingHandled: reports whether the window closing is handled or not on desktops by SetWindowClosingHandled.
  • ebiten.SetWindowClosingHandled: sets whether the window closing is handled or not on desktops. The default state is false.

For actual usages, see examples/windowclosing.

New vsync mode for static applications

(#1556)

In addition to vscyn on and vsync off, Ebiten v2.2 added a new mode: FPSModeVsyncOffMinimum. This indicates that the game doesn't sync with vsync, and the game is updated only when necessary like an input is detected. This is useful for a static application.

  • type ebiten.FPSMode
  • ebiten.SetFPSMode

Append* functions

(#1692, #1705)

Returning a new slice for each call is not efficient. To mitigate this issue, new functions were added to accept a slice and append new elements. These work like the build function append. If the given slice has enough capacity, the functions don't allocate memory. If the given slice is empty or nil, a new slice is created.

  • ebiten.AppendGamepadIDs
  • ebiten.AppendInputChars
  • ebiten.AppendTouchIDs
  • inpututil.AppendJustConnectedGamepadIDs
  • inpututil.AppendJustPressedTouchIDs
  • inpututil.AppendPressedKeys

Performance improvements

  • Memory usage at ReplacePixels was improved (#1681).
  • Allocating color matrices for scaling was improved (#1658).
  • Reduced allocations of color matrix values (#1796).
  • Ensured to free memory for image pixels soon after they were no longer used (#1803).

Other new functions

  • (*ebiten.Image).RGBA64At (#1769).
  • (*audio.Context).NewPlayer: replaces audio.NewPlayer (#1708).
  • (*audio.Context).NewPlayerFromBytes: replaces audioe.NewPlayerFromBytes (#1708).
  • text.DrawWithOptions (#1461).

Others

  • A constant ebiten.SyncWithFPS was added to replace ebiten.UncappedTPS (#1726).
  • audio/mp3, audio/vorbis, audio/wav: Decode and DecodeWithSampleRate now takes io.Reader instead of io.ReadSeeker (#1621).
  • Fixed an issue that dfdx and some functions in non-main function failed to compile in Kage (#1701)
